# リポジトリ運用方針

## 役割分担

- `openai-voice-terminal`:
  - Pi5 実装コード
  - API 連携仕様
  - サーバー挙動変更
- `ATOMS3R-OpenAI-VoicePTT-HTTP`:
  - デバイス実装コード
  - ボタン操作
  - 録音/再生制御
- `openai-voice-terminal-system`:
  - 横断ドキュメント
  - セットアップ/運用/障害対応
  - 変更履歴の要約

## 変更時の基本フロー

1. 変更対象リポジトリを決める
2. 実装リポジトリで変更・検証
3. 仕様変更があれば本リポジトリの docs を更新
4. タグ/リリース時は3リポ整合を確認

## バージョン整合

運用時は以下を常に記録する:

- AtomS3R 側タグ
- Pi5 側コミットまたはタグ
- 本ドキュメントリポジトリ更新コミット

これにより、障害発生時に再現可能な組み合わせを追跡できる。
